# Connect ChatGPT to the Kotlet MCP server

## Prerequisites

- A Kotlet account with an active household.
- The production API must be running.
- If ChatGPT generates a different callback URL, add it to `OAuth:RedirectUris` in `src/backend/Kotlet.Api/appsettings.json` and deploy before connecting.

## Connect

1. Create the app.
2. Sign in through the Kotlet login page when redirected.
3. Confirm that an active household is selected.
4. Test the connection by asking ChatGPT who you are; it should read `kotlet://identity`.
5. Test read access by asking ChatGPT to list recipes or retrieve today's meal plan.

## Reconnect after permission or tool changes

If ChatGPT reports that not all permissions were granted, remove the custom app completely and create it again with the settings above. The **Reconnect** action can retain the previous consent and tool permissions.
